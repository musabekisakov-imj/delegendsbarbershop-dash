# Features

Спецификации фич — пишутся ДО начала реализации, не после.

**Когда писать:** перед любой фичей которая займёт больше 2 часов или затронет больше 3 файлов.

**Как именовать:** `YYYY-MM-DD-название-фичи.md`
Пример: `2026-05-09-recurring-appointments.md`

**Цель:** выровнять понимание с клиентом до того как написана первая строчка кода.
