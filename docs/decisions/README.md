# Architecture Decisions

Сюда идут решения, которые трудно понять задним числом — без контекста.

**Когда писать:** принял нетривиальное решение по архитектуре, стеку, структуре данных,
или отказался от очевидного варианта в пользу менее очевидного.

**Как именовать:** `YYYY-MM-DD-короткое-название.md`
Пример: `2026-05-09-week-view-single-staff.md`

**Не пиши сюда:** мелкие правки UI, баг-фиксы, переименования.
